# 出題語言注意事項（Reference）

供撰寫題敘、官方範例程式或參考解法時對照。以下為**教學與線上評測常見約定**；若 QJudge 平台另有規範，以平台為準。

---

## C++
- **標頭檔**：請使用**標準、明確**的 include，例如 `<iostream>`、`<vector>`、`<string>`、`<algorithm>`、`<cstdint>` 等。  
- **避免**使用單一「統包」非標準標頭（例如競賽常見的 `bits/stdc++.h`）：可攜性與教學清晰度較差，且非所有評測環境皆提供。  
- **輸入／輸出**：題敘階段**不必**要求學生做 fast I/O 或細部 input 優化；以 `cin`/`cout` 或 `scanf`/`printf` 等**清楚、可讀**的寫法即可，除非課程明確在教 I/O 效能。  
- **命名空間**：若範例使用 `using namespace std;`，需在教學上可接受；較嚴謹的課程可示範 `std::` 前綴。  
- **整數範圍**：若數值可能超過 32 位元，題敘或提示中應點出使用 `long long` 或指定型別。

## C
- 使用標準函式庫標頭，例如 `<stdio.h>`、`<stdlib.h>`、`<string.h>`；避免依賴編譔器特有擴充。  
- 與 C++ 相同：**不要求**在題敘中強制 scanf 微優化，除非該題教學目標就是 I/O。

## Python
- 明確**版本**（例如 Python 3.x）若平台有固定版本應引用平台說明。  
- **遞迴深度**：若演算法為深遞迴，需考慮預設遞迴限制，或在題敘／測資上避免觸發（或允許迭代實作）。  
- **浮點數**：若答案為浮點，題敘應規定誤差（絕對／相對）或要求分數輸出格式，避免比對歧義。

## Java
- 常見 OJ 慣例：公開類別名為 `Main`，進入點為 `public static void main(String[] args)`；題敘或範例應與之一致。  
- 注意 `int` 與 `long` 範圍；大整數題需引導使用 `long` 或 `BigInteger`（視教學目標）。

## JavaScript / TypeScript（Node）
- 明確**輸入讀取方式**（例如逐行讀 stdin），因語言慣例多元，題敘應給最小可行範例或連結平台讀取模板。  
- 整數超過 `Number` 安全整數時，需題示使用 `BigInt` 或改以字串處理。

## Go
- 注意 `int` 位元數與平台相關時，大數值題宜寫清楚使用 `int64` 等。  
- 題敘若需自訂 `package`，應與評測環境一致（常見為 `package main`）。

## 本專案目前明確排除的考量
- **C++ input 優化**（例如與 `ios::sync_with_stdio`、緩衝區相關的微調）：出題與題敘撰寫時**不需**列為必備要求或教學重點；若老師希望加入，應視為進階選修並單獨標示。
