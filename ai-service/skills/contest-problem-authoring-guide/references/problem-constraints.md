# 題目邊界設定規則

## 通用規則（適用所有語言）

### 資料結構描述原則

除非用戶明確指定實作細節，否則使用通用的資料結構名稱：

✅ **推薦做法：**
- 使用通用術語：陣列 (Array)、字串 (String)、列表 (List)
- 讓學習者自行選擇適合的實作方式
- 專注於演算法邏輯而非特定實作

❌ **避免做法：**
- 不要指定特定語言的資料結構（除非用戶要求）
- 例如：不要說「使用 vector」、「使用 ArrayList」、「使用 list」

### 範例

**好的題目描述：**
```markdown
給定一個整數陣列，計算所有元素的總和。
```

**不好的題目描述：**
```markdown
給定一個 vector<int>，計算所有元素的總和。
```

## 語言特定規則

### C++ 相關

當題目提到「陣列」時：
- ✅ 學習者可自行選擇使用 `vector`、C-style array、或 `array`
- ❌ 不要在題目中指定 `vector` 除非：
  - 用戶明確要求
  - 題目重點就是練習 `vector` 的使用

### Python 相關

當題目提到「陣列」或「列表」時：
- ✅ 學習者使用 `list` 即可
- ❌ 不需要特別說明使用 `list`（Python 的預設行為）

### Java 相關

當題目提到「陣列」時：
- ✅ 學習者可自行選擇使用 `int[]`、`ArrayList` 等
- ❌ 不要指定使用 `ArrayList` 除非用戶要求

## 何時可以指定實作細節

以下情況可以指定特定資料結構：

1. **用戶明確要求**
   - "請出一題使用 vector 的 C++ 題目"
   - "生成一個練習 ArrayList 的 Java 題目"

2. **題目重點是該資料結構**
   - 例如：練習 Stack、Queue、Hash Table 的使用

3. **需要特定功能**
   - 例如：需要動態調整大小的陣列
   - 這時可以在提示中說明，但不要在題目描述中強制指定

## 檢查清單

生成題目時檢查：
- [ ] 是否使用通用的資料結構名稱？
- [ ] 是否避免了語言特定的實作細節？
- [ ] 如果有指定實作，是否符合用戶需求或題目重點？
