# QJudge i18n 翻譯內容檢視

本次檢視針對 QJudge 專案的多語言翻譯內容進行全面分析，包含翻譯 JSON 檔案、文檔同步狀態，以及前端程式碼中的硬編碼字串檢測。

## 📚 報告文件

本次檢視產生了以下報告文件：

### 1. [綜合檢視報告](./i18n-comprehensive-review-report.md) ⭐ 推薦閱讀
完整的 i18n 翻譯內容分析報告，包含：

- 硬編碼字串掃描結果

### 2. [翻譯 Key 分析報告](./i18n-translation-analysis-report.md)
專注於翻譯 JSON 檔案的分析，包含：
- 各語言翻譯 Key 數量統計
- 缺失翻譯 Key 詳細清單
- 重複 Key 分析

### 3. [硬編碼字串掃描報告](./hardcoded-strings-report.md)
前端程式碼硬編碼字串的完整清單，包含：
- 每個檔案的硬編碼字串位置
- 行號與內容預覽

## 📊 關鍵發現

### ✅ 良好表現
- **4 個主要翻譯檔案**（common.json, problem.json, contest.json, admin.json）在所有語言間完全同步
- **691 個翻譯 key** 在 zh-TW, en, ja, ko 四種語言中保持一致
- **文檔檔案結構完整**，所有語言版本都有 13 個文檔檔案

### ⚠️ 需要改善
1. **docs.json 翻譯不同步**
   - 日語（ja）缺少 35 個 key
   - 韓語（ko）缺少 35 個 key
   
2. **重複的翻譯 Key**
   - 發現 13 個 key 出現在多個檔案中
   
3. **硬編碼中文字串**
   - 52 個檔案包含 598 處硬編碼字串
   - 最嚴重的檔案：ContestLogsPage.tsx (59 處)

## 🔧 使用的分析工具

本次檢視使用了以下自動化工具：

1. **i18n_analysis.py** - 翻譯檔案分析工具
   - 比較所有語言的 key 數量
   - 識別缺失和額外的 key
   - 分析重複的 key

2. **scan_hardcoded_strings.py** - 硬編碼字串掃描工具
   - 掃描 .tsx/.ts 檔案中的中文字串
   - 過濾註解、測試檔案等
   - 生成詳細的位置清單
