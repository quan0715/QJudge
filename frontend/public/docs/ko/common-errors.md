# 자주 발생하는 오류

QJudge 사용 시 자주 발생하는 오류와 해결 방법을 설명합니다.

## 컴파일 오류 (CE)

### 구문 오류

```
error: expected ';' before '}' token
```

**해결 방법**: 세미콜론, 괄호 등의 누락을 확인하세요.

### 선언되지 않은 변수

```
error: 'x' was not declared in this scope
```

**해결 방법**: 모든 변수가 올바르게 선언되었는지 확인하세요.

## 런타임 오류 (RE)

### 배열 범위 초과

```cpp
int arr[10];
arr[10] = 5;  // 오류! 인덱스는 0-9여야 합니다
```

**해결 방법**: 배열 접근이 유효 범위 내인지 확인하세요.

### 0으로 나누기

```cpp
int result = a / b;  // b가 0이면 오류
```

**해결 방법**: 나누기 전에 제수가 0인지 확인하세요.

### 스택 오버플로우

일반적으로 너무 깊은 재귀가 원인입니다.

**해결 방법**:

- 루프 구현으로 변경
- 기본 조건 검사 추가
- 꼬리 재귀 최적화 사용

## 오답 (WA)

### 출력 형식 오류

흔한 문제:

- 추가 공백이나 줄바꿈
- 끝의 줄바꿈 누락
- 대소문자 불일치

**해결 방법**: 출력 형식 요구사항을 주의 깊게 확인하세요.

### 정수 오버플로우

```cpp
int n = 1000000;
int result = n * n;  // 오버플로우!
```

**해결 방법**: 큰 숫자에는 `long long`을 사용하세요.

### 엣지 케이스

자주 놓치는 엣지 케이스:

- n = 0 또는 n = 1
- 빈 문자열
- 음수 입력

## 시간 초과 (TLE)

### 알고리즘 복잡도가 너무 높음

| 데이터 범위 | 권장 복잡도 |
| ----------- | ----------- |
| n ≤ 10      | O(n!)       |
| n ≤ 20      | O(2^n)      |
| n ≤ 500     | O(n³)       |
| n ≤ 5000    | O(n²)       |
| n ≤ 10⁶     | O(n log n)  |
| n ≤ 10⁸     | O(n)        |

### I/O 효율

**C++ 고속화 기법**:

```cpp
ios::sync_with_stdio(false);
cin.tie(nullptr);
```

**피해야 할 것**:

- `endl` (대신 `'\n'` 사용)
- 빈번한 소량 출력

## 메모리 제한 초과 (MLE)

### 배열이 너무 큼

```cpp
int arr[100000000];  // 약 400 MB, 제한 초과 가능
```

**해결 방법**:

- 실제 필요한 배열 크기 평가
- 메모리 효율적인 데이터 구조 사용

### 메모리 누수

동적 할당 사용 시, 대응하는 해제가 있는지 확인하세요.

## 도움이 필요하신가요?

해결할 수 없는 문제가 있다면:

1. 문제의 토론 섹션 확인
2. 대회 중 clarification 기능 사용
3. 동료와 토론 (대회 외)
