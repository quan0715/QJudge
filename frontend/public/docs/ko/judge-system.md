# 채점 시스템

QJudge가 코드를 평가하는 방법을 이해합니다.

## 채점 프로세스

```
코드 제출 → 컴파일 → 테스트 실행 → 출력 비교 → 결과 보고
```

### 1. 컴파일 단계

시스템은 표준 컴파일러를 사용하여 코드를 컴파일합니다:

| 언어   | 컴파일러    | 컴파일 플래그    |
| ------ | ----------- | ---------------- |
| C      | GCC         | `-O2 -std=c11`   |
| C++    | G++         | `-O2 -std=c++17` |
| Python | Python 3.10 | -                |
| Java   | OpenJDK 17  | -                |

### 2. 실행 단계

컴파일 성공 후, 시스템은 여러 테스트 케이스에서 프로그램을 실행하고 다음을 확인합니다:

- 출력이 올바른지
- 실행 시간이 제한 내인지
- 메모리 사용량이 제한 내인지

### 3. 채점 결과

| 결과                  | 약어 | 설명                    |
| --------------------- | ---- | ----------------------- |
| Accepted              | AC   | 모든 테스트 통과        |
| Wrong Answer          | WA   | 출력이 예상값과 다름    |
| Time Limit Exceeded   | TLE  | 실행 시간 제한 초과     |
| Memory Limit Exceeded | MLE  | 메모리 사용량 제한 초과 |
| Runtime Error         | RE   | 실행 중 크래시          |
| Compilation Error     | CE   | 컴파일 실패             |

## 리소스 제한

### 기본 제한

- **시간 제한**: 문제에 따라 다름, 일반적으로 1-2초
- **메모리 제한**: 문제에 따라 다름, 일반적으로 256 MB
- **출력 제한**: 256 MB

### 언어별 조정

| 언어   | 시간 배수 |
| ------ | --------- |
| C/C++  | 1x        |
| Java   | 2x        |
| Python | 3x        |

## 특수 채점

### 부분 점수

일부 문제는 부분 점수를 지원하며, 통과한 테스트 케이스의 비율에 따라 점수가 계산됩니다.

### 스페셜 저지

일부 문제에서는 다음을 위해 스페셜 저지가 사용될 수 있습니다:

- 여러 정답이 있는 문제
- 부동 소수점 오차 허용
- 유연한 출력 형식

## 샌드박스 환경

모든 프로그램은 격리된 샌드박스 환경에서 실행되어 다음을 보장합니다:

- 시스템 보안
- 공정한 채점
- 리소스 격리
