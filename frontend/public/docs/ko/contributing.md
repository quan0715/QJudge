# 기여하기

QJudge 프로젝트에 관심을 가져주셔서 감사합니다! 모든 형태의 기여를 환영합니다.

## 기여 방법

### 문제 보고

버그를 발견하거나 기능 제안이 있는 경우 GitHub Issues에 보고해 주세요:

1. 문제가 아직 보고되지 않았는지 확인
2. 명확한 제목으로 문제 설명
3. 재현 단계 제공 (버그의 경우)
4. 관련 오류 메시지나 스크린샷 첨부

### 코드 제출

1. 프로젝트를 자신의 GitHub 계정으로 포크
2. 개발용 새 브랜치 생성
3. 코드가 프로젝트 가이드라인을 따르는지 확인
4. Pull Request 제출

## 개발 가이드라인

### 코드 스타일

- 프론트엔드는 ESLint + Prettier 사용
- 백엔드는 PEP 8 Python 스타일 가이드 준수
- 커밋 전 `npm run lint` 실행

### 커밋 메시지

명확한 커밋 메시지 사용:

```
feat: 사용자 아바타 업로드 기능 추가
fix: 대회 순위 계산 오류 수정
docs: API 문서 업데이트
refactor: 제출 처리 로직 리팩토링
```

## Pull Request 프로세스

1. 모든 테스트가 통과하는지 확인
2. 관련 문서 업데이트
3. 최소 1명의 메인테이너에게 리뷰 요청
4. CI 체크가 통과할 때까지 대기
5. 병합 후 브랜치 삭제
