# ジャッジシステム

QJudge がコードを評価する方法を理解しましょう。

## ジャッジプロセス

```
コード提出 → コンパイル → テスト実行 → 出力比較 → 結果報告
```

### 1. コンパイルフェーズ

システムは標準のコンパイラを使用してコードをコンパイルします：

| 言語   | コンパイラ  | コンパイルフラグ |
| ------ | ----------- | ---------------- |
| C      | GCC         | `-O2 -std=c11`   |
| C++    | G++         | `-O2 -std=c++17` |
| Python | Python 3.10 | -                |
| Java   | OpenJDK 17  | -                |

### 2. 実行フェーズ

コンパイル成功後、システムはプログラムを複数のテストケースで実行し、以下を確認します：

- 出力が正しいか
- 実行時間が制限内か
- メモリ使用量が制限内か

### 3. ジャッジ結果

| 結果                  | 略称 | 説明                     |
| --------------------- | ---- | ------------------------ |
| Accepted              | AC   | すべてのテストに合格     |
| Wrong Answer          | WA   | 出力が期待値と一致しない |
| Time Limit Exceeded   | TLE  | 実行時間が制限を超過     |
| Memory Limit Exceeded | MLE  | メモリ使用量が制限を超過 |
| Runtime Error         | RE   | 実行中にクラッシュ       |
| Compilation Error     | CE   | コンパイル失敗           |

## リソース制限

### デフォルト制限

- **時間制限**：問題により異なる、通常 1-2 秒
- **メモリ制限**：問題により異なる、通常 256 MB
- **出力制限**：256 MB

### 言語による調整

| 言語   | 時間係数 |
| ------ | -------- |
| C/C++  | 1x       |
| Java   | 2x       |
| Python | 3x       |

## 特殊ジャッジ

### 部分点

一部の問題は部分点をサポートしており、通過したテストケースの割合に基づいて得点が計算されます。

### スペシャルジャッジ

一部の問題では、以下の目的でスペシャルジャッジが使用される場合があります：

- 複数の正解がある問題
- 浮動小数点の誤差許容
- 柔軟な出力形式

## サンドボックス環境

すべてのプログラムは隔離されたサンドボックス環境で実行され、以下を確保します：

- システムセキュリティ
- 公平なジャッジ
- リソースの分離
