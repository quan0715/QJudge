# よくあるエラー

QJudge を使用する際によく発生するエラーとその解決方法について説明します。

## コンパイルエラー (CE)

### 構文エラー

```
error: expected ';' before '}' token
```

**解決方法**：セミコロン、括弧などの欠落をチェックしてください。

### 未宣言の変数

```
error: 'x' was not declared in this scope
```

**解決方法**：すべての変数が正しく宣言されていることを確認してください。

## 実行時エラー (RE)

### 配列の範囲外アクセス

```cpp
int arr[10];
arr[10] = 5;  // エラー！インデックスは 0-9 である必要があります
```

**解決方法**：配列アクセスが有効範囲内であることを確認してください。

### ゼロ除算

```cpp
int result = a / b;  // b が 0 の場合エラー
```

**解決方法**：除算前に除数がゼロでないことを確認してください。

### スタックオーバーフロー

通常、深すぎる再帰が原因です。

**解決方法**：

- ループ実装に変更
- 基底条件のチェックを追加
- 末尾再帰最適化を使用

## 不正解 (WA)

### 出力形式エラー

よくある問題：

- 余分なスペースや改行
- 末尾の改行が欠落
- 大文字小文字の不一致

**解決方法**：出力形式の要件を注意深く確認してください。

### 整数オーバーフロー

```cpp
int n = 1000000;
int result = n * n;  // オーバーフロー！
```

**解決方法**：大きな数には `long long` を使用してください。

### エッジケース

よく見落とされるエッジケース：

- n = 0 または n = 1
- 空文字列
- 負数入力

## 時間制限超過 (TLE)

### アルゴリズムの計算量が高すぎる

| データ範囲 | 推奨計算量 |
| ---------- | ---------- |
| n ≤ 10     | O(n!)      |
| n ≤ 20     | O(2^n)     |
| n ≤ 500    | O(n³)      |
| n ≤ 5000   | O(n²)      |
| n ≤ 10⁶    | O(n log n) |
| n ≤ 10⁸    | O(n)       |

### I/O 効率

**C++ 高速化テクニック**：

```cpp
ios::sync_with_stdio(false);
cin.tie(nullptr);
```

**避けるべき**：

- `endl`（代わりに `'\n'` を使用）
- 頻繁な少量出力

## メモリ制限超過 (MLE)

### 配列が大きすぎる

```cpp
int arr[100000000];  // 約 400 MB、制限超過の可能性
```

**解決方法**：

- 実際に必要な配列サイズを評価
- よりメモリ効率の良いデータ構造を使用

### メモリリーク

動的割り当てを使用する場合、対応する解放があることを確認してください。

## ヘルプが必要ですか？

解決できない問題に遭遇した場合：

1. 問題のディスカッションセクションを確認
2. コンテスト中は clarification 機能を使用
3. クラスメートと議論（コンテスト外）
