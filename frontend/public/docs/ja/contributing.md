# コントリビュート

QJudge プロジェクトへの関心をお寄せいただきありがとうございます！あらゆる形式の貢献を歓迎します。

## 貢献方法

### 問題の報告

バグを見つけた場合や機能の提案がある場合は、GitHub Issues で報告してください：

1. 問題がまだ報告されていないことを確認
2. 明確なタイトルで問題を説明
3. 再現手順を提供（バグの場合）
4. 関連するエラーメッセージやスクリーンショットを添付

### コードの提出

1. プロジェクトを自分の GitHub アカウントにフォーク
2. 開発用の新しいブランチを作成
3. コードがプロジェクトのガイドラインに従っていることを確認
4. Pull Request を提出

## 開発ガイドライン

### コードスタイル

- フロントエンドは ESLint + Prettier を使用
- バックエンドは PEP 8 Python スタイルガイドに従う
- コミット前に `npm run lint` を実行

### コミットメッセージ

明確なコミットメッセージを使用：

```
feat: ユーザーアバターアップロード機能を追加
fix: コンテストランキング計算エラーを修正
docs: API ドキュメントを更新
refactor: 提出処理ロジックをリファクタリング
```

## Pull Request プロセス

1. すべてのテストが通過することを確認
2. 関連ドキュメントを更新
3. 少なくとも 1 人のメンテナーにレビューを依頼
4. CI チェックが通過するまで待機
5. マージ後にブランチを削除
