# Contributing

Thank you for your interest in contributing to QJudge! We welcome all forms of contributions.

## Ways to Contribute

### Reporting Issues

If you find a bug or have a feature suggestion, please report it on GitHub Issues:

1. Check if the issue has already been reported
2. Use a clear title to describe the issue
3. Provide reproduction steps (for bugs)
4. Include relevant error messages or screenshots

### Submitting Code

1. Fork the project to your GitHub account
2. Create a new branch for development
3. Ensure code follows project guidelines
4. Submit a Pull Request

## Development Guidelines

### Code Style

- Frontend uses ESLint + Prettier for code formatting
- Backend follows PEP 8 Python style guide
- Run `npm run lint` before committing

### Commit Messages

Use clear commit messages:

```
feat: add user avatar upload feature
fix: fix contest ranking calculation error
docs: update API documentation
refactor: refactor submission processing logic
```

### Branch Naming

- `feature/` - New features
- `fix/` - Bug fixes
- `docs/` - Documentation updates
- `refactor/` - Code refactoring

## Pull Request Process

1. Ensure all tests pass
2. Update relevant documentation
3. Request review from at least one maintainer
4. Wait for CI checks to pass
5. Delete branch after merging

## Need Help?

If you encounter issues while contributing:

- Ask questions in Issues
- Contact project maintainers
