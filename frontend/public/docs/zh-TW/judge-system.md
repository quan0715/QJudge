# 評測系統說明

了解 QJudge 的程式評測機制。

## 評測流程

```
提交程式碼 → 編譯 → 執行測試 → 比對輸出 → 回報結果
```

### 1. 編譯階段

系統會使用標準編譯器編譯您的程式碼：

| 語言   | 編譯器      | 編譯參數         |
| ------ | ----------- | ---------------- |
| C      | GCC         | `-O2 -std=c11`   |
| C++    | G++         | `-O2 -std=c++17` |
| Python | Python 3.10 | -                |
| Java   | OpenJDK 17  | -                |

### 2. 執行階段

編譯成功後，系統會使用多組測試資料執行您的程式，並檢查：

- 輸出是否正確
- 執行時間是否在限制內
- 記憶體使用是否在限制內

### 3. 評測結果

| 結果                  | 縮寫 | 說明               |
| --------------------- | ---- | ------------------ |
| Accepted              | AC   | 程式通過所有測試   |
| Wrong Answer          | WA   | 輸出與預期不符     |
| Time Limit Exceeded   | TLE  | 執行時間超過限制   |
| Memory Limit Exceeded | MLE  | 記憶體使用超過限制 |
| Runtime Error         | RE   | 程式執行時發生錯誤 |
| Compilation Error     | CE   | 程式編譯失敗       |

## 資源限制

### 預設限制

- **時間限制**：依題目設定，通常為 1-2 秒
- **記憶體限制**：依題目設定，通常為 256 MB
- **輸出限制**：256 MB

### 語言調整

不同程式語言可能有不同的時間係數：

- C/C++：1x
- Java：2x
- Python：3x

## 特殊評測

### 部分給分

某些題目支援部分給分，依通過的測試點比例計算得分。

### 特殊比對

某些題目可能使用特殊比對器 (Special Judge)，用於：

- 多解問題
- 浮點數誤差容許
- 格式較為靈活的輸出

## 沙盒環境

所有程式都在隔離的沙盒環境中執行，確保：

- 系統安全性
- 評測公平性
- 資源隔離
