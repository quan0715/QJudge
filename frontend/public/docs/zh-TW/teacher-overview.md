# 教師功能總覽

本文介紹 QJudge 系統中教師角色可使用的功能。

## 權限說明

教師帳號擁有以下權限：

- 建立與管理競賽
- 新增與編輯題目
- 查看學生提交紀錄
- 管理競賽參與者

## 競賽管理

### 建立競賽

1. 點擊導覽列的「建立競賽」
2. 填寫競賽基本資訊：
   - 競賽名稱
   - 開始與結束時間
   - 競賽類型（公開/私人）
3. 新增競賽題目
4. 設定計分方式

### 競賽類型

| 類型     | 說明                           |
| -------- | ------------------------------ |
| 公開競賽 | 所有使用者皆可參加             |
| 私人競賽 | 需要密碼才能參加，適合課堂作業 |

### 計分模式

- **ACM/ICPC 模式**：以解題數為主，罰時為輔
- **OI 模式**：依測試點給分，累計總分

## 題目管理

### 新增題目

1. 進入「題目管理」頁面
2. 點擊「新增題目」
3. 填寫題目資訊：
   - 題目標題（支援多語言）
   - 題目描述
   - 輸入輸出格式
   - 測試資料
   - 時間/記憶體限制

### 測試資料格式

測試資料需包含：

- 輸入檔案 (`.in`)
- 預期輸出檔案 (`.out`)

### 題目難度

建議依據以下標準設定難度：

| 難度   | 適用情境             |
| ------ | -------------------- |
| Easy   | 基礎語法、簡單迴圈   |
| Medium | 常見演算法、資料結構 |
| Hard   | 進階演算法、複雜問題 |

## 查看提交紀錄

教師可以：

- 查看所有學生的提交紀錄
- 下載學生程式碼
- 查看詳細評測結果

## 問答管理 (Clarifications)

競賽期間，教師可以：

- 回覆學生的問題
- 發布公告給所有參賽者
- 更新題目說明

## 最佳實踐

1. **提前測試**：競賽開始前完整測試所有題目
2. **設定合理時限**：考慮不同語言的效能差異
3. **準備範例**：提供清晰的範例測試資料
4. **監控競賽**：即時關注問答區的問題
