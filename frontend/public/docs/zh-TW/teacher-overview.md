# 教師指南：建立與管理考試

> 文件狀態：2026-03-09

歡迎使用 QJudge 平台。本指南旨在協助教師快速掌握從「建立考卷」、「題目編輯」、「監考處理」到「成績批改」的完整工作流程。

## 核心工作流程

建立一場成功的測驗通常包含以下五個階段：

### 1. 建立考試 (Create Exam)
選擇適當的考試類型（Coding Test 或 Paper-like Test），設定考試時間與參加權限。
- **相關指南**：[建立考試與題目設定](/docs/zh-TW/create-exam)

### 2. 題目編輯 (Question Authoring)
使用 Markdown 與 LaTeX 建立豐富的題幹，設定配分與自動批改規則。
- **支援題型**：單選、多選、是非、簡答、問答。
- **相關指南**：[題目編輯器詳細說明](/docs/zh-TW/create-exam#3-建立並編輯考試題目)

### 3. 監考與人工介入 (Proctoring & Intervention)
即時監控考生的作答狀態，處理突發狀況（如：意外鎖定解鎖、手動增加考生、強制繳卷）。
- **關鍵功能**：全螢幕監控、違規自動鎖定、管理員手動解鎖。
- **相關指南**：[監考與人工介入處理](/docs/zh-TW/exam-proctoring)

### 4. 成績批改 (Grading)
利用自動批改節省時間，並對主觀題進行人工閱卷與評分。
- **相關指南**：[考試批改與成績發布](/docs/zh-TW/exam-grading)

### 5. 成績統計與檢視 (Statistics & Review)
分析全班得分分布，檢視各題目的錯誤率與學生的具體作答內容。
- **相關指南**：[考試作答統計與檢視](/docs/zh-TW/exam-statistics)

---

## 教學實務建議

1. **考前演練**：建議在正式考試前，先建立一個「測試用競賽」讓學生熟悉登入與作答流程。
2. **防弊設定**：對於正式考試，務必開啟「防作弊模式」並設定合理的「最大違規次數」。
3. **題目凍結**：一旦有學生開始作答，題目內容與順序將會被凍結。若需修改，請在考前確認所有細節。
4. **備份題目**：利用 JSON 匯出功能備份您的題庫，方便日後跨課程復用。
