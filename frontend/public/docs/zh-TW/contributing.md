# 如何貢獻

感謝您對 QJudge 專案的興趣！我們歡迎各種形式的貢獻。

## 貢獻方式

### 回報問題

如果您發現了 Bug 或有功能建議，請在 GitHub Issues 中回報：

1. 確認問題尚未被回報過
2. 使用清晰的標題描述問題
3. 提供重現步驟（如果是 Bug）
4. 附上相關的錯誤訊息或截圖

### 提交程式碼

1. Fork 專案到您的 GitHub 帳號
2. 建立新的分支進行開發
3. 確保程式碼符合專案規範
4. 提交 Pull Request

## 開發規範

### 程式碼風格

- 前端使用 ESLint + Prettier 進行程式碼格式化
- 後端遵循 PEP 8 Python 風格指南
- 提交前請執行 `npm run lint` 檢查

### Commit 訊息

請使用清晰的 Commit 訊息：

```
feat: 新增使用者頭像上傳功能
fix: 修復競賽排名計算錯誤
docs: 更新 API 文件
refactor: 重構提交處理邏輯
```

### 分支命名

- `feature/` - 新功能
- `fix/` - Bug 修復
- `docs/` - 文件更新
- `refactor/` - 程式碼重構

## Pull Request 流程

1. 確保所有測試通過
2. 更新相關文件
3. 請求至少一位維護者審核
4. 等待 CI 檢查通過
5. 合併後刪除分支

## 需要協助？

如果您在貢獻過程中遇到問題，可以：

- 在 Issue 中提問
- 聯繫專案維護者
