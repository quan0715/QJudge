# 批改、統計與回饋

QJudge 的價值不只在於收答案，也在於把作答資料轉成教學回饋。考試或作業結束後，教師可以透過自動評測、人工批改與統計資料，判斷學生是否真正理解課程內容。

## 批改流程

### 1. 先分辨題型

不同題型適合不同批改方式：

| 題型 | 建議批改方式 | 注意事項 |
| :--- | :--- | :--- |
| 程式題 | 自動評測為主 | 檢查測資是否公平且覆蓋邊界 |
| 單選 / 多選 / 是非 | 自動計分 | 考後確認答案設定是否正確 |
| 簡答題 | 人工或半人工 | 先建立可接受答案範圍 |
| 問答題 | 人工批改 | 使用評分規準提高一致性 |

### 2. 處理客觀題

客觀題批改前請確認：

- 正確答案是否設定正確。
- 配分是否符合考卷設計。
- 是否有題目需要送分或調整分數。
- 是否有學生因系統或題目問題受影響。

### 3. 處理主觀題

主觀題建議先建立簡短評分規準，再開始批改。規準可以包含：

- 核心觀念是否正確。
- 是否有完整推理或步驟。
- 是否使用正確術語。
- 是否有明顯抄襲或不合理相似。

若有多位助教共同批改，建議先抽樣 5 至 10 份答案共同校準標準。

## 使用統計找出教學重點

考後統計不只是排名工具，更適合用來找出下一堂課要補強的內容。

### 建議觀察指標

| 指標 | 可回答的問題 |
| :--- | :--- |
| 題目平均分 | 哪些概念全班普遍不熟？ |
| 題目通過率 | 哪些題目可能太難或敘述不清？ |
| 測資失敗分布 | 學生常在哪些邊界條件出錯？ |
| 提交次數 | 學生是否反覆嘗試但無法突破？ |
| 未提交名單 | 是否有學生需要額外關懷？ |

相關文件：[考試作答統計與檢視](#/docs/exam-statistics)、[考試批改與成績發布](#/docs/exam-grading)

## 回饋學生

### 個別回饋

個別回饋適合用於：

- 主觀題批改。
- 程式設計風格建議。
- 多次提交仍未通過的學生。
- 需要補救教學的學生。

回饋應盡量具體，例如「第 3 題沒有處理空陣列」比「邏輯錯誤」更有幫助。

### 全班回饋

全班回饋適合處理普遍問題。建議整理：

- 最常錯的 3 個觀念。
- 代表性錯誤答案。
- 正確解法的關鍵步驟。
- 下一次作業或考試會如何延伸。

## 成績發布前檢查

發布成績前，請確認：

- 所有需人工批改的題目都已完成。
- 題目配分總和正確。
- 送分、扣分與補分規則已套用。
- 異常事件已有處理紀錄。
- 是否要先讓助教複查高風險題目。

## 把結果帶回教學

建議在下一堂課安排 10 至 20 分鐘講解作業或考試結果：

1. 先說明全班整體表現。
2. 挑選 2 至 3 個常見錯誤。
3. 示範如何從錯誤訊息或測資推回問題。
4. 給學生一題短練習確認是否已修正觀念。

## 長期維護題目品質

每次作業或考試後，請回頭更新題庫：

- 修正容易誤解的題目敘述。
- 補充缺少的範例。
- 增加邊界測資。
- 標記題目難度與適用週次。
- 記錄這題在課堂上的教學效果。

這些整理會讓下一次開課更穩定，也能減少助教重複處理相同問題。
