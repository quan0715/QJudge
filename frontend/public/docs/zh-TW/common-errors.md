# 常見錯誤

本文整理了使用 QJudge 時常見的錯誤與解決方法。

## 編譯錯誤 (Compilation Error)

### 語法錯誤

```
error: expected ';' before '}' token
```

**解決方法**：檢查程式碼中是否有遺漏的分號、括號等。

### 未宣告的變數

```
error: 'x' was not declared in this scope
```

**解決方法**：確保所有變數都有正確宣告。

### 標頭檔缺失

```
fatal error: bits/stdc++.h: No such file or directory
```

**解決方法**：改用標準標頭檔，如 `<iostream>`、`<vector>` 等。

## 執行錯誤 (Runtime Error)

### 陣列越界

```cpp
int arr[10];
arr[10] = 5;  // 錯誤！索引應為 0-9
```

**解決方法**：確保陣列存取在有效範圍內。

### 除以零

```cpp
int result = a / b;  // 若 b 為 0 會發生錯誤
```

**解決方法**：在除法前檢查除數是否為零。

### 堆疊溢位

通常由過深的遞迴造成。

**解決方法**：

- 改用迴圈實作
- 增加基礎情況檢查
- 使用尾遞迴優化

## 答案錯誤 (Wrong Answer)

### 輸出格式錯誤

常見問題：

- 多餘的空格或換行
- 缺少換行符號
- 大小寫不符

**解決方法**：仔細核對題目的輸出格式要求。

### 整數溢位

```cpp
int n = 1000000;
int result = n * n;  // 溢位！
```

**解決方法**：使用 `long long` 處理大數。

### 邊界情況

常見遺漏的邊界情況：

- n = 0 或 n = 1
- 空字串
- 負數輸入

## 超時 (Time Limit Exceeded)

### 演算法複雜度過高

| 資料範圍 | 建議複雜度 |
| -------- | ---------- |
| n ≤ 10   | O(n!)      |
| n ≤ 20   | O(2^n)     |
| n ≤ 500  | O(n³)      |
| n ≤ 5000 | O(n²)      |
| n ≤ 10⁶  | O(n log n) |
| n ≤ 10⁸  | O(n)       |

### I/O 效率

**C++ 加速技巧**：

```cpp
ios::sync_with_stdio(false);
cin.tie(nullptr);
```

**避免使用**：

- `endl`（改用 `'\n'`）
- 頻繁的小量輸出

## 記憶體超限 (Memory Limit Exceeded)

### 陣列過大

```cpp
int arr[100000000];  // 約 400 MB，可能超限
```

**解決方法**：

- 評估實際需要的陣列大小
- 使用更節省空間的資料結構

### 記憶體洩漏

使用動態配置時，確保有對應的釋放。
