# 題庫管理 (Question Bank)

題庫模組是 QJudge 存放教學資產的核心，旨在協助教師有條理地管理各類題目、維護版本，並在多個教室與考試間重複利用這些題目。

---

## 一、 題庫分類

根據題目性質，題庫分為兩大類別：

### 1. 程式題庫 (Coding Bank)
專門存放程式設計（Coding）題目，支援設定自動評測腳本、範例測試資料與執行環境限制。
- **特點**：每一題都與特定程式語言環境綁定，支援 YAML 匯入與進階測試流程設定。

### 2. 筆試題庫 (Exam Bank)
存放選擇、是非、簡答、問答等各類測驗題目。
- **特點**：支援多種子題型，並能快速匯出為 JSON 或匯入至各種考試競賽。

---

## 二、 題庫管理流程

### 1. 建立題庫
1. 於主選單進入「題庫」頁面，點擊「新增題庫」。
2. 設定題庫名稱、描述，並選擇「題庫類別」（程式題庫或筆試題庫）。
3. **可見度設定**：
   - **私有 (Private)**：僅限擁有者與受邀管理員查看。
   - **公開 (Public)**：經過審核後，所有教師皆可從此題庫中選題。

### 2. 編輯題目與版本管理
- 題庫中的題目支援版本控制。當您修改題目內容（如題幹、測資）後，系統會自動儲存為新的版本。
- 當競賽引用此題目時，您可以選擇鎖定在特定版本，或追蹤最新版本。

### 3. 公開題庫與審核機制
- 若您希望分享您的題庫，可發起「公開申請」。
- 系統管理員審核通過後，該題庫將出現在「公共題庫區」，供全平台教師引用。

---

## 三、 在考試中引用題庫

### 1. 連結題庫題目
在建立考試題目時（無論是 Coding 或 Paper-like Exam），您可以選擇「從題庫匯入」：
1. 瀏覽您擁有的私有題庫或平台提供的公共題庫。
2. 勾選欲加入的題目，點擊「確認引用」。

### 2. 複本與引用 (Copy vs Reference)
- **直接匯入**：系統會建立一個獨立的題目副本存入該場競賽。您在競賽中修改內容，不會影響原始題庫。
- **引用綁定**：題目與題庫保持連結。若題庫題目修正，您可選擇一鍵同步更新競賽中的題目。

---

## 四、 匯入與匯出 (Bulk Actions)

- **YAML 匯入**：針對程式題庫，您可以依照 `problem.yaml` 格式進行批量上傳。
- **JSON 匯入/匯出**：針對筆試題庫，支援標準 JSON 格式的備份與遷移。詳細規格請參考 [Exam 題目 JSON 匯入/匯出](/docs/zh-TW/exam-json-import-export)。

---

## 五、 教學實務建議

1. **依學科分類**：建議為每一門學科建立獨立的題庫（例如：「C++ 基礎題庫」、「計算機概論題庫」）。
2. **標記難度**：在題目描述或標籤中註記題目難度，方便日後隨機挑選題目組成試卷。
3. **版本保留**：在重大期中考前，建議將引用的題庫版本手動鎖定，避免因意外修改題庫內容而影響正在進行的考試。
