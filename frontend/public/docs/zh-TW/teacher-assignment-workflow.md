# 作業與練習導入流程

作業與練習是最適合持續導入 QJudge 的場景。相較於正式考試，平時練習容錯較高，也能讓學生透過反覆提交建立解題習慣。教師可以從每週一場小練習開始，再逐步發展成完整作業制度。

## 建議流程

### 1. 先決定活動目的

建立作業前，先判斷這次活動的目標：

| 活動目的 | 建議設定 | 評量重點 |
| :--- | :--- | :--- |
| 課堂即時練習 | 時間短、允許多次提交 | 是否理解剛教完的概念 |
| 課後作業 | 開放數天、題目較完整 | 完成度與正確性 |
| 小考 | 時間固定、題目較少 | 獨立解題能力 |
| 補救練習 | 不限次數、可重做 | 補足基本觀念 |

### 2. 從題庫準備題目

建議將常用題目放在題庫，而不是只存在單一作業中。這樣可以：

- 下學期重複使用。
- 讓助教先測試題目。
- 保留版本紀錄。
- 將同一題放入不同班級或不同活動。

相關文件：[題庫管理](#/docs/question-bank)

### 3. 在教室內建立活動

若這個作業屬於某門課，建議從教室內建立競賽或考試，讓活動自動綁定教室成員。這樣可以減少學生手動輸入密碼或找錯活動的機率。

建議命名方式：

| 類型 | 名稱範例 |
| :--- | :--- |
| 課堂練習 | W03 課堂練習：迴圈與陣列 |
| 課後作業 | HW02：排序與搜尋 |
| 小考 | Quiz 01：C++ 基本語法 |

### 4. 設定提交規則

平時作業通常建議允許多次提交，讓學生可以根據評測結果修正。若作業用於成績評量，需要先說明：

- 是否採最高分。
- 是否採最後一次提交。
- 截止後是否允許補交。
- 延遲提交如何扣分。

### 5. 發布活動公告

公告應包含：

- 作業名稱。
- 截止時間。
- 是否計分。
- 提交次數規則。
- 遇到題目問題時的回報方式。

## 課堂練習設計

課堂練習適合設計成 10 至 25 分鐘的小任務。建議題目不要太長，重點是讓教師能快速看到全班理解狀況。

### 操作建議

1. 課堂講解一個觀念。
2. 發布 1 至 2 題練習。
3. 讓學生現場提交。
4. 教師從提交狀況觀察卡住的比例。
5. 針對常見錯誤立即講解。

## 課後作業設計

課後作業可以比課堂練習更完整，但仍應避免一次放入太多不相關的題目。建議每份作業對應一個主要學習目標。

### 題目描述建議

每題至少包含：

- 題目背景。
- 輸入格式。
- 輸出格式。
- 範例輸入與輸出。
- 限制條件。
- 評分或測資說明。

### 測資建議

測資應覆蓋：

- 基本案例。
- 邊界條件。
- 常見錯誤。
- 效能壓力。

若學生常因題意不清而大量錯誤，應優先修正題目敘述，而不是只增加測資。

## 使用統計改善教學

作業結束後，建議教師查看：

- 哪些題目通過率最低。
- 哪些測資錯誤最多。
- 哪些學生完全沒有提交。
- 哪些學生提交很多次但仍無法通過。

這些資料可以轉化成下一堂課的講解內容。例如，如果多數學生在邊界條件失敗，下一堂課可以安排 10 分鐘專門討論測資設計與邊界分析。

## 常見導入問題

### 學生只靠測資猜答案

可搭配隱藏測資、開放題目討論、要求提交解題說明，或在課堂中抽問思路。

### 作業量難以控制

先用少題高品質作業。對初學者而言，一份作業 2 至 4 題通常比 8 至 10 題更容易形成有效回饋。

### 助教批改負擔太重

客觀程式題交給自動評測，主觀題只保留必要的反思題或設計題。若需要人工批改，請事先準備評分規準。
