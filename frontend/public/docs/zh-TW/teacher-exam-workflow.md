# 正式考試導入流程

正式考試需要比平時練習更完整的規劃。除了題目本身，教師還需要處理考前演練、考試模式、防弊規則、學生異常狀況與成績發布。建議每門課第一次正式考試前，至少安排一次模擬考。

## 考前規劃

### 1. 決定考試型態

QJudge 常見考試型態如下：

| 型態 | 適用情境 | 建議功能 |
| :--- | :--- | :--- |
| 程式上機考 | 程式設計、資料結構、演算法 | 程式題、自動評測、提交紀錄 |
| 紙筆型線上考 | 選擇題、簡答題、問答題 | Paper Exam、人工批改、成績統計 |
| 混合型測驗 | 程式題搭配概念題 | 競賽題目與 Paper Exam 分開發布 |

### 2. 建立模擬考

正式考試前，建議建立一場 10 至 20 分鐘模擬考。模擬考不需要難，但要涵蓋正式考試會出現的操作：

- 進入考試。
- 通過考前檢查。
- 啟用全螢幕或考試模式。
- 完成提交。
- 遇到鎖定時向監考人員求助。

### 3. 確認考試規則

考前應明確公告：

- 考試開始與結束時間。
- 可使用的程式語言或工具。
- 是否允許查閱教材。
- 是否允許離開全螢幕。
- 被鎖定後如何處理。
- 網路或設備異常如何回報。

## 建立正式考試

### 1. 從教室建立

若考試屬於一門課，建議從教室內建立考試。這樣考試會自動綁定教室成員，學生從教室入口即可找到考試。

相關文件：[建立考試與題目](#/docs/create-exam)

### 2. 設定時間

請確認：

- 開始時間是否正確。
- 結束時間是否包含緩衝。
- 是否需要提前開放學生進入等待。
- 補考或延長時間是否有規劃。

### 3. 設定防弊與作答限制

正式考試通常建議啟用考試模式，但不要在第一次使用時設定過於嚴格的規則。若學生尚未熟悉平台，過度嚴格的鎖定規則會增加監考負擔。

建議做法：

| 場景 | 建議設定 |
| :--- | :--- |
| 第一次正式使用 | 先完成模擬考，再啟用合理違規次數 |
| 小班上機考 | 可使用較嚴格的全螢幕與鎖定規則 |
| 大班考試 | 設定清楚的自動解鎖或助教分工流程 |
| 遠距考試 | 事先公告設備、網路與瀏覽器要求 |

相關文件：[監考與人工介入處理](#/docs/exam-proctoring)、[Exam Pre-check 與防弊時序](#/docs/exam-precheck-anticheat)

## 考試當天流程

### 考前 15 分鐘

1. 教師與助教登入管理頁。
2. 確認考試設定、題目與時間。
3. 開啟監考或管理畫面。
4. 提醒學生登入正確帳號。
5. 處理尚未加入教室或名單異常的學生。

### 考試開始後

監考人員應觀察：

- 是否有學生尚未進入考試。
- 是否有大量學生卡在考前檢查。
- 是否有學生被鎖定。
- 是否有題目敘述或測資異常回報。

若有題目問題，請先判斷是否影響公平性。已開始考試後修改題目需要非常謹慎，必要時可透過公告補充說明。

### 考試結束前

建議提醒學生：

- 儲存或提交最後答案。
- 確認提交紀錄。
- 不要在最後一分鐘才第一次提交。

## 異常狀況處理

### 學生被鎖定

先確認原因，再決定是否手動解鎖。對於正式考試，建議每次解鎖都保留簡短紀錄，例如「瀏覽器當機重開」、「投影通知切出全螢幕」。

### 學生網路中斷

確認學生是否能重新登入並回到考試。若需要補時間，請依課程規則處理，避免臨場標準不一致。

### 題目或測資錯誤

若錯誤影響全班，建議立即公告並在考後統一調整分數。若只影響少數人，仍應記錄事件與處理方式。

## 考後收尾

考試結束後，請先確認：

- 提交紀錄是否完整。
- 是否有未處理的鎖定或異常事件。
- 是否有需要人工批改的題目。
- 是否要延後發布成績。
