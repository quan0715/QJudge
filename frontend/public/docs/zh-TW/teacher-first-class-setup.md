# 第一堂課前設定

第一次把 QJudge 帶進課堂時，目標不是一次啟用所有功能，而是讓學生順利完成「加入教室、進入練習、提交答案、查看結果」這條最短路徑。只要第一堂課能穩定完成這個流程，後續導入作業與考試會容易很多。

## 課前準備清單

### 1. 建立教室

1. 登入 QJudge。
2. 進入「教室」頁面。
3. 點擊「新增教室」。
4. 填寫課程名稱、課程代碼與簡短描述。
5. 確認教室建立後可以看到邀請資訊。

建議命名方式：

| 欄位 | 範例 |
| :--- | :--- |
| 教室名稱 | 115 學年度上學期 資料結構 |
| 課程代碼 | CS201 |
| 描述 | 課堂練習、作業與考試皆於此教室發布 |

相關文件：[教室管理](#/docs/classroom-management)

### 2. 加入助教或共同教師

若課程有助教，建議在第一堂課前就加入教室。助教至少要能協助：

- 確認學生是否成功加入。
- 處理學生登入與加入碼問題。
- 測試練習題是否能正常提交。
- 在正式考試時協助監考與解鎖。

### 3. 建立入門練習

第一場練習應該非常簡單。它的目的不是評量學生能力，而是確認環境與操作流程。

建議練習內容：

| 題型 | 建議題目 | 目的 |
| :--- | :--- | :--- |
| 程式題 | 輸出 Hello QJudge | 測試提交與編譯 |
| 選擇題 | 課程規則確認 | 測試紙筆題作答 |
| 簡答題 | 輸入學生自我介紹 | 測試文字作答 |

### 4. 準備學生說明

第一堂課建議投影或公告以下資訊：

- QJudge 網址。
- 學生登入方式。
- 教室邀請碼或加入連結。
- 本次練習是否計分。
- 遇到問題時要如何回報。

## 課堂操作流程

### 步驟一：確認所有學生能登入

先不要急著進入題目。請學生完成登入，並確認畫面能正常顯示。若課程使用校園帳號或 Google 登入，建議先提醒學生使用學校指定帳號。

### 步驟二：加入教室

讓學生使用邀請碼加入教室。教師或助教應在教室成員列表確認人數是否接近實際到課人數。

如果有學生無法加入，先確認：

1. 是否登入了正確帳號。
2. 邀請碼是否輸入正確。
3. 學生是否已加入其他錯誤教室。

### 步驟三：進入練習活動

請學生從教室內找到本次練習。第一次導入時，建議使用清楚的活動名稱，例如「第 1 週 QJudge 操作練習」。

### 步驟四：完成一次提交

請所有學生完成至少一次提交。對程式題而言，學生需要知道：

- 哪裡撰寫程式。
- 如何選擇語言。
- 如何提交。
- 如何查看編譯錯誤、測資結果或通過狀態。

### 步驟五：收斂問題

課程最後保留 5 分鐘整理問題。建議把問題分成三類：

| 類型 | 例子 | 後續處理 |
| :--- | :--- | :--- |
| 帳號問題 | 無法登入、登入錯帳號 | 課後個別處理 |
| 操作問題 | 找不到教室、找不到提交按鈕 | 下次課堂再示範 |
| 題目問題 | 題意不清、測資不合理 | 修改題目或補充說明 |

## 課後檢查

第一堂課後，教師應確認：

- 教室成員名單是否完整。
- 是否有學生使用錯誤帳號。
- 練習活動是否收到大部分學生提交。
- 常見問題是否需要補進課程公告。

## 下一步

完成第一堂課導入後，可以開始把 QJudge 用於平時作業與課堂練習。請接著閱讀：[作業與練習導入流程](#/docs/teacher-assignment-workflow)。
