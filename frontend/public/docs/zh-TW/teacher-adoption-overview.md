# 教師課堂導入與建立考試

這份文件協助教師把 QJudge 導入課堂，並建立第一場可讓學生實際參與的考試。建議先用一場低風險的練習型考試完成流程演練，再把同一套流程用在正式期中、期末或上機測驗。

本文專注在兩件事：

1. 教師如何在課堂中導入 QJudge。
2. 教師如何建立一場考試並讓學生順利作答。

## 導入前先決定使用情境

建立考試前，請先決定這場活動在課堂中的用途。不同用途會影響考試設定、時間安排與監考強度。

| 使用情境 | 建議做法 | 注意事項 |
| :--- | :--- | :--- |
| 第一次課堂演練 | 建立不計分或低配分考試 | 目標是確認學生會登入、加入教室、進入考試與提交 |
| 平時小考 | 題數少、時間短、規則清楚 | 避免第一次就開啟過於嚴格的防弊設定 |
| 期中 / 期末考 | 先做模擬考，再建立正式考試 | 需要事先安排助教分工、異常處理與成績發布流程 |
| 上機程式測驗 | 使用程式題與自動評測 | 題目敘述、範例測資與語言限制要先測過 |
| 紙筆型線上測驗 | 使用選擇、是非、簡答或問答題 | 主觀題需要預留人工批改時間 |

如果是第一次使用 QJudge，請先不要把第一場活動設為高風險正式考試。先安排一場 10 到 20 分鐘的「系統熟悉測驗」，讓學生完成一次完整流程。

## 建議導入流程

### 第一步：建立教室

教室通常對應一門課程。建議所有課堂考試都從教室出發，這樣學生名單、考試入口與後續成績紀錄會比較一致。

建議設定：

- 教室名稱：使用學期與課程名稱，例如「115 學年度上學期 資料結構」。
- 課程描述：簡短說明這個教室會用於練習、作業或考試。
- 助教權限：若有助教，請先加入教室並確認助教可以協助管理考試。

完成後，請準備好教室邀請碼或加入方式，供第一堂課或考前公告使用。

相關文件：[教室管理](#/docs/classroom-management)

### 第二步：規劃第一場考試

建立考試前，先把考試規則寫清楚。最少需要決定：

- 考試名稱。
- 開始與結束時間。
- 是否計分。
- 題型：程式題、選擇題、是非題、簡答題或問答題。
- 是否需要監考或防弊設定。
- 學生遇到登入、網路或瀏覽器問題時要找誰處理。

第一次導入時，建議使用簡單題目，例如：

| 題型 | 範例 | 目的 |
| :--- | :--- | :--- |
| 程式題 | 輸入兩個整數並輸出總和 | 確認學生會提交程式碼 |
| 選擇題 | 課程規則確認 | 確認學生會作答與送出答案 |
| 簡答題 | 輸入一句課程心得或姓名確認 | 確認文字作答流程 |

這場測驗的重點不是評量能力，而是確認所有學生都能完成操作。

### 第三步：建立考試

建議從教室內建立考試，讓考試與班級成員自動對齊。

基本流程：

1. 進入目標教室。
2. 在教室管理或考試列表中建立新的競賽 / 考試。
3. 填寫考試名稱、說明、開始時間與結束時間。
4. 選擇考試類型。
5. 儲存後進入考試管理頁面。

考試名稱建議使用清楚格式：

- 「第 1 週 QJudge 操作練習」
- 「W03 小考：迴圈與陣列」
- 「期中考模擬測驗」
- 「115-1 資料結構期中考」

相關文件：[建立考試與題目](#/docs/create-exam)

### 第四步：加入題目

建立考試後，請加入題目並檢查題目呈現是否正確。

每一題至少要確認：

- 題目敘述清楚。
- 配分正確。
- 選擇題或是非題有設定正確答案。
- 程式題有範例輸入與輸出。
- 程式題測資已由教師或助教試跑。
- 主觀題有參考答案或批改標準。

第一次導入時，題目數量不宜太多。建議先用 1 到 3 題完成流程演練。

### 第五步：設定考試規則

考試規則會影響學生能否順利進入與完成作答。請至少檢查：

- 考試時間是否正確。
- 學生是否能從教室看到考試。
- 是否需要加入密碼。
- 是否開啟考試模式或防弊設定。
- 是否允許學生多次提交。
- 是否要在考後立即顯示結果。

若這是第一次課堂導入，建議先使用較低干擾的設定，讓學生熟悉平台。正式考試再依課程需求調整防弊與監考規則。

### 第六步：先做教師端檢查

發布給學生前，教師或助教應先完整走過一次：

1. 確認考試入口可見。
2. 確認題目順序與配分正確。
3. 確認作答畫面可以正常開啟。
4. 程式題至少提交一次範例解。
5. 紙筆題至少作答一次並確認答案可儲存。
6. 若有啟用考試模式，確認考前檢查流程可完成。

這一步可以減少正式上課時才發現設定錯誤的風險。

## 課堂當天建議流程

### 考前 10 分鐘

請學生完成：

- 登入 QJudge。
- 加入教室。
- 找到本次考試。
- 確認瀏覽器與網路狀態。

教師或助教應同步檢查：

- 教室成員人數是否接近實際到課人數。
- 是否有學生登入錯帳號。
- 是否有學生找不到教室或考試。

### 考試開始

請先確認大部分學生都已進入考試，再正式開始計時或提醒學生作答。

若是第一次導入，可以請學生完成一題後暫停一下，確認大家都知道：

- 如何作答。
- 如何提交。
- 如何查看提交或儲存狀態。
- 遇到問題時如何舉手或回報。

### 考試進行中

教師與助教應觀察：

- 是否有人無法進入考試。
- 是否有人卡在考前檢查。
- 是否有學生被鎖定。
- 是否有題目敘述或測資問題。

如果是正式考試，任何臨時處理都建議保留簡短紀錄，例如「學生瀏覽器當機」、「重新登入」、「監考老師手動解鎖」。

### 考試結束

考試結束後，請確認：

- 學生是否都已提交或完成交卷。
- 是否有未處理的異常狀況。
- 是否有主觀題需要人工批改。
- 是否要立即發布成績，或先由教師檢查後再發布。

相關文件：[批改、統計與回饋](#/docs/teacher-grading-feedback)

## 正式考試前的最低檢查清單

在期中、期末或高風險考試前，請至少完成以下檢查：

- 已建立教室並確認學生名單。
- 已建立一場模擬考或低風險練習。
- 學生已知道如何登入、進入考試與提交。
- 考試題目、配分與時間已由教師或助教檢查。
- 程式題測資已試跑。
- 主觀題已有批改標準。
- 助教知道如何處理學生無法進入、被鎖定或需要協助的情況。
- 教師已決定成績何時發布。

## 不建議的導入方式

- 第一次使用 QJudge 就直接舉辦正式期中考。
- 沒有模擬考就開啟嚴格防弊設定。
- 題目尚未試跑就發布給學生。
- 只公告考試連結，沒有確認學生是否已加入教室。
- 考試規則只口頭說明，沒有寫在公告或課堂投影片中。
