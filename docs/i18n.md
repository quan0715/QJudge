# i18n 多國語系開發指南

本專案使用 `react-i18next` 進行多國語系管理。目前支援繁體中文 (`zh-TW`)、英文 (`en`)、日文 (`ja`) 與韓文 (`ko`)。

## 檔案結構
i18n 資源檔案位於 `frontend/src/i18n/locales/`，按語言代碼分層：
- `zh-TW/`: 繁體中文（主要參考源 / Source of Truth）
- `en/`: 英文
- `ja/`: 日文
- `ko/`: 韓文

每個語言目錄下包含多個命名空間 (Namespace) JSON 檔案：
- `common.json`: 共用文案、選單、偏好設定、分頁、按鈕等。
- `contest.json`: 競賽介面、監考流程、成績冊。
- `problem.json`: 題目列表、編輯器說明、測試結果。
- `landing.json`: 官網首頁推廣文案。
- `classroom.json`: 教室管理相關。
- `chatbot.json`: AI 助教對話介面。
- `admin.json`: 系統後台管理。
- `docs.json`: 幫助文件、說明。

## 開發流程

### 1. 新增或修改 Key
**請務必先修改 `zh-TW` 目錄下的 JSON 檔案。** `zh-TW` 是本專案的主要參考來源。

### 2. 同步 Key 到其他語言
修改 `zh-TW` 後，執行以下命令自動將新 Key 同步至 `en`, `ja`, `ko`：
```bash
cd frontend
npm run sync:i18n
```
**該命令會執行以下動作：**
- 遍歷所有語言目錄，確保 Key 結構完全對齊。
- 若其他語言缺少某個 Key，將自動補齊並使用 `zh-TW` 作為預設內容（方便後續翻譯人員識別需要翻譯的項目）。
- 自動對 JSON 檔案進行字母順序排序，保持 git diff 清潔。

### 3. 檢查同步狀態 (CI)
在 CI 流程或提交前，可以執行檢查命令來驗證語系檔案是否完全同步：
```bash
cd frontend
npm run check:i18n
```

### 4. 在前端程式碼中使用
```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  // 指定 Namespace 為 "common"
  const { t } = useTranslation("common"); 
  
  return (
    <div>
      {/* 使用點號分隔路徑 */}
      <h1>{t("nav.dashboard")}</h1>
      
      {/* 支援帶參數的翻譯 */}
      <p>{t("dashboard.welcomeBack", { name: "User" })}</p>
    </div>
  );
}
```

## 翻譯規範與注意事項
- **排序**：JSON 檔案應保持 Key 排序。請務必使用 `npm run sync:i18n` 進行自動排序與同步，不要手動調整順序。
- **佔位符**：使用 `{{variable}}` 格式。
- **命名規範**：
  - 按鈕請放入 `common.json` 的 `button` 物件下（例：`t("button.save")`）。
  - 通用操作請放入 `common.json` 的 `action` 或 `status` 下。
- **Source of Truth**：當 `en`/`ja`/`ko` 有額外 Key 而 `zh-TW` 沒有時，同步腳本會將該 Key 回補到 `zh-TW`。因此，建議開發時統一從 `zh-TW` 開始操作。
