## 🚀 快速開始

### 系統需求

- **瀏覽器**: Chrome、Firefox、Safari、Edge（最新版本）
- **網路**: 穩定的網際網路連接
- **裝置**: 電腦或平板（建議使用電腦以獲得最佳體驗）

### 存取平台

1. 開啟瀏覽器並前往：**https://nycu-coding-lab.quan.wtf**
2. 首次使用請先註冊帳號
3. 已有帳號請直接登入

## 👤 註冊與登入

### 註冊新帳號

1. 點擊首頁右上角的「註冊」按鈕
2. 填寫註冊資訊：
   - **使用者名稱**: 3-30 個字元，只能包含英文字母、數字、底線
   - **Email**: 有效的電子郵件地址（用於接收通知）
   - **密碼**: 至少 8 個字元，建議包含大小寫字母、數字和特殊符號
   - **確認密碼**: 再次輸入密碼以確認
3. 閱讀並同意服務條款
4. 點擊「註冊」按鈕
5. 系統會發送驗證郵件到您的信箱
6. 點擊郵件中的驗證連結完成註冊

### 登入系統

1. 點擊首頁右上角的「登入」按鈕
2. 輸入您的 Email 和密碼
3. （可選）勾選「記住我」以保持登入狀態
4. 點擊「登入」按鈕

### 忘記密碼

1. 在登入頁面點擊「忘記密碼？」
2. 輸入您註冊時使用的 Email
3. 系統會發送密碼重設連結到您的信箱
4. 點擊郵件中的連結設定新密碼

## 📚 瀏覽題目

### 進入題庫

1. 登入後點擊導覽列的「題目」或「題庫」
2. 您將看到所有可用的題目列表

### 題目列表功能

#### 篩選題目

- **難度篩選**: 點擊「簡單」、「中等」、「困難」標籤
- **標籤篩選**: 選擇特定演算法或資料結構標籤
- **搜尋**: 在搜尋框輸入關鍵字搜尋題目

#### 題目資訊

每個題目會顯示：
- **題目編號**: 如 P001, P002
- **題目標題**: 題目的名稱
- **難度**: 🟢 簡單 / 🟡 中等 / 🔴 困難
- **通過率**: 成功通過的提交比例
- **狀態**: 
  - ✅ 已通過（Accepted）
  - ❌ 已嘗試但未通過
  - 空白表示尚未嘗試

#### 排序題目

點擊欄位標題可以排序：
- 按題目編號
- 按難度
- 按通過率
- 按最近更新時間

## 💻 解題流程

### 1. 閱讀題目

點擊題目標題進入題目詳細頁面，包含：

#### 題目描述
- 題目的背景說明
- 需要解決的問題
- 輸入輸出格式說明
- 限制條件（時間限制、記憶體限制）

#### 範例測試

每題會提供數個範例：
```
輸入範例 1:
1 2

輸出範例 1:
3

說明:
1 + 2 = 3
```

#### 提示區域
- 演算法提示
- 解題思路建議
- 注意事項

### 2. 選擇程式語言

在題目頁面的程式碼編輯器上方，選擇您要使用的程式語言：

支援的語言：
- 🔵 **C++**: C++17 標準
- 🟡 **Python**: Python 3.11
- 🔴 **Java**: Java 17
- 🟢 **JavaScript**: Node.js 18

### 3. 撰寫程式碼

#### 使用線上編輯器

系統提供功能完整的線上程式碼編輯器（Monaco Editor）：

**編輯器功能**：
- ✨ 語法高亮顯示
- 🔍 自動完成（IntelliSense）
- 📏 程式碼格式化
- 🔢 行號顯示
- 🎨 多種主題可選

**快捷鍵**：
- `Ctrl/Cmd + S`: 儲存程式碼
- `Ctrl/Cmd + /`: 註解/取消註解
- `Ctrl/Cmd + F`: 搜尋
- `Ctrl/Cmd + H`: 搜尋並取代
- `Tab`: 縮排
- `Shift + Tab`: 取消縮排

#### 程式碼範本

系統會根據您選擇的語言提供基本範本：

**C++ 範本**：
```cpp
#include <iostream>
using namespace std;

int main() {
    // 在這裡撰寫您的程式碼
    
    return 0;
}
```

**Python 範本**：
```python
# 在這裡撰寫您的程式碼


```

**Java 範本**：
```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        // 在這裡撰寫您的程式碼
    }
}
```

### 4. 測試程式碼

提交前可以先自行測試：

1. 點擊「執行」或「測試」按鈕
2. 輸入測試資料
3. 查看程式輸出
4. 確認結果是否正確

**自訂測試資料**：
```
輸入:
5 3

預期輸出:
8
```

### 5. 提交程式碼

確認程式碼無誤後：

1. 點擊「提交」按鈕
2. 系統會開始評測您的程式
3. 評測過程中會顯示即時狀態

### 6. 查看評測結果

#### 評測狀態

提交後可能出現的狀態：

| 狀態 | 說明 |
|------|------|
| ⏳ **Pending** | 等待評測中 |
| ⚙️ **Running** | 正在執行中 |
| ✅ **Accepted (AC)** | 完全正確 |
| ❌ **Wrong Answer (WA)** | 輸出錯誤 |
| ⏱️ **Time Limit Exceeded (TLE)** | 執行超時 |
| 💾 **Memory Limit Exceeded (MLE)** | 記憶體超限 |
| 💥 **Runtime Error (RE)** | 執行錯誤 |
| 🔧 **Compilation Error (CE)** | 編譯錯誤 |

#### 詳細評測資訊

點擊提交記錄可以查看：
- **測試點結果**: 每個測試點的通過情況
- **執行時間**: 程式執行所花費的時間
- **記憶體使用**: 程式使用的記憶體大小
- **錯誤訊息**: 如果有錯誤，會顯示詳細資訊

#### 範例測試結果
```
測試點 #1: ✅ Accepted (10ms, 2.1MB)
測試點 #2: ✅ Accepted (15ms, 2.3MB)
測試點 #3: ❌ Wrong Answer (12ms, 2.2MB)
測試點 #4: ⏱️ Time Limit Exceeded
測試點 #5: ✅ Accepted (18ms, 2.5MB)

總分: 60/100
```

### 7. 除錯與改進

如果未能通過所有測試：

1. **查看錯誤類型**: 了解問題所在
2. **檢查範例測試**: 確認範例都能通過
3. **考慮邊界情況**: 
   - 最小值、最大值
   - 空輸入
   - 特殊情況
4. **優化演算法**: 
   - 減少時間複雜度
   - 降低記憶體使用
5. **重新提交**: 修改後再次提交

## 🏆 參加競賽

### 瀏覽競賽

1. 點擊導覽列的「競賽」
2. 查看可用的競賽列表

競賽狀態：
- 🟢 **即將開始**: 尚未開始的競賽
- 🔴 **進行中**: 正在進行的競賽
- ⚫ **已結束**: 已經結束的競賽

### 註冊競賽

1. 點擊想要參加的競賽
2. 閱讀競賽規則和說明
3. 點擊「註冊參加」按鈕
4. 如果需要密碼，輸入老師提供的競賽密碼
5. 確認註冊

### 競賽開始前

- **確認競賽時間**: 注意競賽開始和結束時間
- **準備環境**: 確保網路穩定、電腦狀況良好
- **閱讀規則**: 了解競賽規則、計分方式、注意事項
- **提前登入**: 建議提前 10-15 分鐘登入系統

### 競賽進行中

#### 進入競賽

1. 在競賽開始時間，點擊「進入競賽」
2. 進入競賽專用頁面

#### 競賽介面

競賽頁面包含：
- **題目列表**: 競賽中的所有題目
- **剩餘時間**: 倒數計時器
- **即時排名**: 當前的排名情況
- **公告區**: 老師發布的公告和澄清

#### 解題策略

1. **快速瀏覽**: 先瀏覽所有題目，評估難度
2. **先易後難**: 從簡單的題目開始
3. **獲取分數**: 確保拿到基本分數
4. **時間分配**: 合理分配解題時間
5. **檢查提交**: 提交前仔細檢查程式碼

#### 提交限制

注意競賽可能有：
- **提交次數限制**: 每題最多提交 N 次
- **罰時規則**: 錯誤提交會增加罰時
- **冷卻時間**: 相同題目提交間隔時間

#### 查看排名

點擊「排名」標籤查看：
- **排名**: 您當前的排名
- **解題數**: 通過的題目數量
- **總分**: 累積分數
- **罰時**: 總罰時（如適用）
- **其他參賽者**: 其他人的表現

#### 競賽規則

**常見計分規則**：
- **ACM 規則**: 以解題數為主，罰時為輔
  - 解題數多者排名靠前
  - 解題數相同時，罰時少者排名靠前
  - 罰時 = 通過時間 + 錯誤提交次數 × 罰時（通常 20 分鐘）

- **OI 規則**: 以總分為主
  - 總分高者排名靠前
  - 總分相同時，最後提交時間早者排名靠前
  - 部分通過可以獲得部分分數

### 競賽結束後

- **查看最終排名**: 競賽結束後可以查看完整排名
- **檢視題解**: 老師可能會公布參考答案
- **總結經驗**: 回顧競賽表現，找出改進空間

## 💡 解題技巧

### 基本技巧

1. **仔細閱讀題目**: 
   - 理解題目要求
   - 注意輸入輸出格式
   - 確認限制條件

2. **分析範例測試**:
   - 手動模擬範例
   - 理解解題邏輯
   - 找出解題規律

3. **思考演算法**:
   - 選擇合適的演算法
   - 評估時間複雜度
   - 考慮空間複雜度

4. **程式碼實作**:
   - 先寫出基本架構
   - 逐步完成細節
   - 適當加入註解

5. **測試驗證**:
   - 用範例測試
   - 設計極端案例
   - 檢查邊界條件

### 常見錯誤

#### Wrong Answer (WA)

可能原因：
- 演算法邏輯錯誤
- 未考慮所有情況
- 輸出格式不正確
- 精度問題（浮點數）

解決方法：
- 重新檢查演算法
- 測試更多案例
- 確認輸出格式
- 注意換行、空格

#### Time Limit Exceeded (TLE)

可能原因：
- 演算法時間複雜度過高
- 存在無窮迴圈
- 重複計算太多

解決方法：
- 優化演算法
- 使用更高效的資料結構
- 減少不必要的計算
- 加入記憶化/動態規劃

#### Memory Limit Exceeded (MLE)

可能原因：
- 陣列開太大
- 遞迴太深
- 記憶體洩漏

解決方法：
- 優化空間使用
- 改用迭代方法
- 釋放不用的記憶體

#### Runtime Error (RE)

可能原因：
- 陣列越界
- 除以零
- 遞迴太深導致堆疊溢位
- 空指標/空引用

解決方法：
- 檢查陣列索引
- 確認除法運算
- 限制遞迴深度
- 確認指標有效性

#### Compilation Error (CE)

可能原因：
- 語法錯誤
- 函式未宣告
- 變數未定義
- 標頭檔未引入

解決方法：
- 檢查語法
- 確認函式宣告
- 檢查變數命名
- 加入必要的標頭檔

### 解題心態

- 🎯 **保持耐心**: 解題需要時間和練習
- 💪 **持續學習**: 從錯誤中學習成長

- 📚 **累積經驗**: 多做題目累積經驗

## ❓ 常見問題

### 帳號相關

**Q: 我忘記密碼了怎麼辦？**  
A: 在登入頁面點擊「忘記密碼」，系統會發送重設連結到您的 Email。

**Q: 我沒有收到驗證郵件？**  
A: 請檢查垃圾郵件夾，或在個人設定頁面重新發送驗證郵件。

**Q: 可以更改使用者名稱嗎？**  
A: 使用者名稱一旦註冊後通常無法更改，請在註冊時謹慎選擇。

### 題目相關

**Q: 為什麼我的程式在本地可以執行，但提交後顯示錯誤？**  
A: 可能是輸入輸出格式不符、環境差異、或未考慮所有測試案例。請仔細閱讀題目要求。

**Q: 範例測試都通過了，為什麼還是 WA？**  
A: 範例測試只是部分測試案例，還有隱藏的測試案例。請考慮更多邊界情況。

**Q: 如何知道隱藏測試案例是什麼？**  
A: 系統通常不會公開隱藏測試案例，但會顯示錯誤發生在哪個測試點。您需要自行思考可能的案例。

### 競賽相關

**Q: 競賽中可以使用網路搜尋嗎？**  
A: 這取決於競賽規則。一般學習性質的競賽允許查閱資料，但考試性質的競賽通常禁止。請遵守老師的規定。

**Q: 競賽中提交錯誤會有什麼影響？**  
A: 在 ACM 規則下，錯誤提交會增加罰時。在 OI 規則下，只計算最高分的提交。

**Q: 競賽時間不夠怎麼辦？**  
A: 建議先完成有把握的題目，確保基本分數，再挑戰較難的題目。

### 技術相關

**Q: 為什麼我的程式一直顯示 Pending？**  
A: 可能是評測系統繁忙，請稍等片刻。如果長時間未評測，請聯絡管理員。

**Q: 可以使用哪些標準函式庫？**  
A: 各語言的標準函式庫都可以使用，但不能使用第三方函式庫。

**Q: 如何提高程式執行效率？**  
A: 
- 選擇適當的演算法和資料結構
- 減少不必要的運算
- 注意輸入輸出效率（如 C++ 使用 `cin.tie(0)`, `ios::sync_with_stdio(false)`）
