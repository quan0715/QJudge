# Rubric: Define Access Transparency, Location Transparency, and Replication Transparency

**Total: 3 points** (每項 1 分，共三項)

---

## Access Transparency (1 pt)
- **核心概念**：本地與遠端資源使用**相同/一致的操作**進行存取。
- 關鍵詞：identical operations, same way/method, 相同操作/方式。
- 給分：
  - 1 pt：表達出「本地/遠端存取方式相同」的正確概念。
  - 0 pt：未作答、明顯錯誤（如解釋為「自由存取」、「有無權限」、「直接連線」等無關概念）。
- 寬容：不要求 identical 精確用詞，概念正確即可。

## Location Transparency (1 pt)
- **核心概念**：使用者存取資源時**不需要知道資源的實體/實際位置**。
- 關鍵詞：without knowing physical/actual location, 不知位置, 隱藏位置。
- 給分：
  - 1 pt：表達出「不需知道實際位置」的正確概念。
  - 0 pt：未作答、明顯錯誤（如解釋為「知道位置」、「定位資源」、「傳送資料到本地端」等）。
- 寬容：不要求 exact wording。

## Replication Transparency (1 pt)
- **核心概念**：系統存在多個副本/replicas 以提升可靠度或效能，但**使用者不知情**。
- 關鍵詞：multiple copies/replicas, reliability/performance, user unaware / hidden from user.
- 給分：
  - 1 pt：表達出「有多個副本存在但使用者不知道」的正確概念。只提副本存在或只提使用者不知情但概念正確亦可。
  - 0 pt：未作答、明顯錯誤（如解釋為「複製不影響他人」、「重複傳送」、「可複製檔案到本地」等）。
- 寬容：只要核心概念到位即可。

---

## Reason 政策
- 滿分 (3/3)：reason 留空。
- 非滿分 (0-2/3)：reason 簡述哪些項目有誤/缺漏。
