# Rubric — Multilevel Feedback Queue Scheduling Parameters

## 題目
What are the parameters of Multilevel Feedback Queue Scheduling? (List at least three)

## 滿分
3 分

## 參考答案（共 5 個合法參數）
a. number of queues（佇列數量）
b. scheduling algorithms for each queue（每條佇列所用的排程演算法）
c. methods used to determine when to upgrade a process（決定何時升級行程的方法）
d. methods used to determine when to demote a process（決定何時降級行程的方法）
e. methods used to determine which queue a process will enter when that process needs service（決定行程首次進入哪條佇列的方法）

## 評分標準

| 答對參數數量 | 分數 |
|------------|------|
| 3 個（含）以上正確 | 3 |
| 恰好 2 個正確 | 2 |
| 恰好 1 個正確 | 1 |
| 0 個正確 / 未作答 | 0 |

## 判斷原則
- 學生只需列出至少 3 個即可得滿分；超過 3 個亦計 3 分（上限）。
- 措辭不需完全相同，概念正確即可。
  - 例：「number of queues」寫成「how many queues」視為正確。
  - 例：「upgrade / promote」互用可接受；「demote / lower priority」互用可接受。
- 重複列同一概念（e.g. upgrade 寫兩次）只計 1 個。
- 與參考答案 a–e 明顯不相關的敘述不計分。
- reason 政策：滿分（3 分）→ reason 留空；非滿分 → 必須填 reason 說明扣分原因。
