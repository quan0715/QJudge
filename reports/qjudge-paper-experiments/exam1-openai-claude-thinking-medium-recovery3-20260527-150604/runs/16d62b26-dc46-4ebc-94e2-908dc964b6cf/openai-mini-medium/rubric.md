# 評分準則

- 滿分：3 分
- 題目重點：比較 Many-to-One 與 One-to-One 多執行緒模型，並說明哪一種較能善用多核心處理器及原因。

## 配分

1. **Many-to-One 定義**（1 分）
   - 說明多個使用者層執行緒對映到單一核心執行緒。

2. **One-to-One 定義**（1 分）
   - 說明每個使用者層執行緒對映到一個核心執行緒。

3. **多核心利用與原因**（1 分）
   - 明確指出 **One-to-One** 較能善用多核心，因為多個執行緒可被核心平行排程執行。
   - 若寫 Many-to-One 無法有效利用多核心、同時只能在單一核心執行，也可得分。

## 扣分原則

- 只提到其中一種模型但未比較：依內容完整度酌扣。
- 只說「One-to-One 比較好」但沒說明原因：最多給 1 分。
- 說反或混淆兩種模型：扣對應定義與原因分。
- 提到阻塞、平行執行、核心排程等關鍵原因，可視為有掌握第 3 點。

## 理想答案摘要

- Many-to-One：多個 user threads 對映到一個 kernel thread。
- One-to-One：每個 user thread 對映到一個 kernel thread。
- One-to-One 較適合多核心，因為可讓多個執行緒同時在不同核心上平行執行；Many-to-One 受限於只有一個 kernel thread，無法真正並行。
