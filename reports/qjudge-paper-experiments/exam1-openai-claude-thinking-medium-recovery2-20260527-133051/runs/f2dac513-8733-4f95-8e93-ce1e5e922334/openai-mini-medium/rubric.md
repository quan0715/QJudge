# 批改準則：Demand Paging

- 題目滿分：2 分
- 評分重點：
  1. 需說明 demand paging 的概念：頁面只在需要時才載入記憶體，通常是在發生 page fault 時載入，不是程式一開始全載入。
  2. 需說明常用原因：
     - 減少初始載入時間
     - 節省實體記憶體，避免載入未使用頁面
     - 可支援超過實體記憶體大小的程式

- 配分建議：
  - 2 分：概念與用途/優點都說清楚，語意正確。
  - 1 分：只答對其中一部分，或描述大致正確但不完整。
  - 0 分：明顯錯誤、空白、或與 demand paging 無關。

- 可接受表述：
  - 「需要時才載入頁面」、「page fault 後才把頁面載入」
  - 「減少 memory usage」、「降低 startup time」、「可跑大於 RAM 的程式」

- 主要扣分情況：
  - 把 demand paging 說成預先全部載入
  - 只談虛擬記憶體但沒提 demand/on-demand 的核心概念
  - 只談一般 paging，未提需要時載入
