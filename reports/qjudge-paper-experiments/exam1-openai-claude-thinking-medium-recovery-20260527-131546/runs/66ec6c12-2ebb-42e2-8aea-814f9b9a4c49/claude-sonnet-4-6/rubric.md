# Rubric — fork + execl 程式輸出題

## 題目
```c
main() {
 fork();
 execl("/bin/echo", "echo", "hello", 0);
 printf("done\n");
}
```
問：這段程式的執行結果是什麼？

## 滿分
2 分

## 參考答案
"hello" 被印出兩次；"done" 永遠不會被印出。
（execl 會取代目前行程的記憶體映像，故 printf 不會執行。）

---

## 評分準則

| 分數 | 條件 |
|------|------|
| 2 | 同時答對以下兩點：① "hello" 印出**兩次** ② "done" **不會**被印出（並能隱含或明說因為 execl 取代 process image） |
| 1 | 只答對其中一點：① 提到 "hello" 印出兩次，但未提及 "done" 的輸出情形；**或** ② 提到 "done" 不會印出，但未提及 "hello" 印出兩次；**或** ③ 提到 fork 產生子行程、兩個行程各執行 execl，但答案不完整 |
| 0 | 兩點皆錯、空白、或只說 "hello" 印一次等根本錯誤 |

## 評分細節說明
- 關鍵概念 1：`fork()` 產生兩個行程（parent + child），兩個行程都會呼叫 `execl`。
- 關鍵概念 2：`execl` 成功後不會返回，後續的 `printf("done\n")` 不會執行。
- 答案不需要用精確的技術詞彙，只要表達正確語意即可。
- 若回答含有明顯錯誤（如說 fork 不影響輸出、done 也會印出等），視錯誤嚴重程度給 0 或 1 分。
