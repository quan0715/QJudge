# 評分準則：UNIX inode 最大檔案大小

## 題目
UNIX inode，12 個 direct blocks + 1 個 single indirect block，block size = 8KB，address size = 4 bytes，計算最大檔案大小。

## 正確答案
- Direct: 12 × 8KB = **96KB**
- Single indirect: (8KB / 4B) = 2048 個 block pointers → 2048 × 8KB = **16,384KB = 16MB**
- **總計: 96KB + 16MB = 16,480KB ≈ 16.096MB**

## 配分：滿分 2 分

### 2 分（滿分）
符合下列任一：
- 明確寫出 **96KB + 16MB**（或 16MB + 96KB），無重大錯誤
- 給出 **16,480KB**（或 ~16.096MB、~16MB）
- 給出正確的計算式（如 `(12 + 2K) × 8KB` 或 `12×8KB + 2048×8KB`）且結果合理
- 僅寫「~16MB」或「16MB」（近似值也算對）
- 僅寫正確的式子（未算出最終數值），但式子本身正確且構成完整答案
- 大小寫、空格、逗號、約等號等格式不拘

> reason：滿分者 reason **留空**（依 SOP 預設政策）

### 1 分（部分分數）
符合下列任一：
- Direct 正確（96KB）但 indirect 部分數值錯誤（如 32MB、4MB、16GB、98KB）
- Indirect 正確但 direct 部分數值錯誤（如 24KB、16K）
- 計算式概念正確但有明顯的計算錯誤（如 `2KB×8KB` 不合理乘積、單位混用）
- 只給出部分答案（如只寫 96KB 或 16MB 其中一項，且無近似修飾）
- 答案矛盾或不一致（如 352KB 又寫 32MB）
- 有單位誤植但不影響理解（如 16G 應為 16M）

### 0 分（不給分）
符合下列任一：
- 答案與正確答案完全無關（如 56、4KB、2MB、1024TB）
- 概念完全錯誤
- 空白或亂寫
